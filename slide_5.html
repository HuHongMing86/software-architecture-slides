Five Core Components Form the Space-Based Topology

Decoupling transaction processing from database operations

These components work together to enable high throughput and near-infinite elasticity by removing the central database bottleneck.

Component Role
Processing Unit Contains application code and in-memory data grid
Virtualized Middleware Manages and coordinates processing units
Data Pump Asynchronously sends updates to the database
Data Writer Receives messages from data pumps and updates the database
Data Reader Reads data from the database and delivers it to processing units on startup
