Concert Ticketing and Online Auctions Are Ideal Use Cases

Both scenarios require high performance, high scalability, and high elasticity – the three defining strengths of Space-Based Architecture.
Concert Ticketing System
  • User load surges from hundreds to thousands (or tens of thousands) when tickets go on sale.
  • Seat availability must be updated continuously with minimal latency.
  • The Deployment Manager pre-starts processing units before ticket sales begin, placing them on standby.
Online Auction System
  • Unpredictable user and request load spikes when auctions start.
  • Multiple processing units can be dedicated to individual auctions for data consistency.
  • Asynchronous data pumps send bidding data to other processing units (bid history, analytics, audit) with low latency.
