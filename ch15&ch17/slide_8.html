Data Pumps Enable Asynchronous Database Updates

Decoupling processing units from direct database writes

  • Mechanism: Always asynchronous, typically implemented using messaging (e.g., message queues).
  • Benefits: Supports guaranteed delivery, FIFO ordering, and decoupling between processing units and data writers.
  • Granularity: Multiple data pumps exist, each dedicated to a specific domain or subdomain (e.g., Customer Profile, Customer Wishlist).
  • Contract: Each data pump has an associated contract specifying operations (add, delete, update) and data format (JSON, XML).
When a processing unit receives a request and updates its cache, it becomes the "update owner" and is responsible for sending the update through the data pump.
