Microservices Topology – Small, Single-Purpose Services

  • Services: Much smaller than in other distributed architectures (e.g., SOA). Each service contains everything needed for independent operation, including its own database.
  • API Layer: An optional but common layer between consumers and services. Used for proxying, naming services, and service discovery. Should NOT be used as a mediator or orchestration engine.
  • Distributed Nature: Each service runs in its own process (physical machine → VM → container). Solves resource sharing and isolation problems inherent in multi-tenant infrastructure.
