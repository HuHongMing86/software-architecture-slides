Data Writers and Data Readers Form the Data Abstraction Layer

Together, they form a data abstraction layer where the replicated cache schema in processing units may differ from the underlying database table structure. This allows incremental database changes without impacting processing units.
Data Writer
Accepts messages from data pumps and updates the database. Can be domain-based (one writer per domain) or dedicated per data pump.
Data Reader
Reads data from the database and sends it to processing units via a reverse data pump.Invoked only when: All instances crash, all instances are redeployed, or archived data is needed.
