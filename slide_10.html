Data Collisions Arise from Replication Latency

When using active/active replicated caches, data collisions can occur due to replication delay between instances.

Example Calculation

With 5 instances, 20 updates/sec, 50,000 rows, and 100ms latency, there are 72,000 updates/hour with only 14.4 potential collisions (0.02%).

Impact of Latency

Reducing replication latency from 100ms to 1ms drops potential collisions to just 0.1 per hour (0.0002%).

Key Insight: Cache size is inversely proportional to collision probability; all other factors are directly proportional.

Collision Probability
CP = N × (UR² / S) × RL
  • N = Number of instances
  • UR = Update rate (ms)
  • S = Cache size (rows)
  • RL = Replication latency (ms)
