Traditional Web Topology Creates a Scalability Triangle

The bottleneck inevitably shifts to the database layer

  • Web Server Layer: The easiest and cheapest to scale horizontally. It forms the wide top of the triangle.
  • Application Server Layer: More complex and expensive to scale. The bottleneck shifts here after web servers scale.
  • Database Server Layer: The hardest and most expensive to scale. It becomes the ultimate limiting factor, forming the narrow bottom of the triangle.
  • The Problem: This "triangle topology" limits the overall system throughput. Space-Based Architecture was designed specifically to solve this fundamental problem.
